## まとめ

### 那須野

- AWSの [S3 Files](https://aws.amazon.com/jp/blogs/news/launching-s3-files-making-s3-buckets-accessible-as-file-systems/) を試した。
  - 内部的に Amazon EFS の技術をベースにしており、NFS プロトコル（v4.1+）を介して S3 バケットをマウントする仕組み、らしい。
  - いくつか準備が必要（はまりポイント）：
    - S3バケットのバージョニングを有効化しておく
    - IAM 権限に `AmazonS3FilesFullAccess` 等のポリシーが必要
    - S3 で「ファイルシステム」を作成する → ファイルシステムとマウントターゲットの作成完了まで1分程度  
    ![スクリーンショット 2026-04-10 16.09.44](https://hackmd.io/_uploads/Syuo-4InWe.png)
    - EC2作成時のストレージ設定で、EBSのルートボリュームに加えて、「ファイルシステム」/ 「S3ファイル−新規」を指定できるようになる  
    ![スクリーンショット 2026-04-10 16.14.24](https://hackmd.io/_uploads/rJIWXVIhZl.png)
    - S3ファイルシステムのマウントターゲットのセキュリティグループ設定で、inbound 2049/tcp を許可
      - ソースはEC2インスタンスのセキュリティグループIDを指定
  - マウントコマンド: `mount -t s3files fs-xxxxxxxx /mnt/s3/fs1`
      - `amazon-efs-utils` パッケージに `/sbin/mount.s3files` が新たに追加されている
      - df の見え方はこんな感じ ↓ （8エクサバイト!!）
      ```
      ~# df -h /mnt/s3/fs1
      Filesystem      Size  Used Avail Use% Mounted on
      127.0.0.1:/     8.0E     0  8.0E   0% /mnt/s3/fs1
      ```
  - EC2上で書き込んだファイルがS3のWebUIで見えるまでに2分程度かかった
      - レイテンシではなくスループット重視の思想か
